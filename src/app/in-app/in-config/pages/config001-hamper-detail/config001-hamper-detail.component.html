Đây là phiên bản mới đã tổ chức lại cấu trúc folder

  1. Tất cả các component sẽ được khai báo trong laymodule
  2. Đặt tên component theo số: mới số do một bạn đảm nhiệm

    Ví dụ: config0001-hamper-detail là bản hamper do Hoàng làm thì các bạn sẽ test bài tập của hoàng trong component này

    Ví dụ: config0002-hamper-detail là bản hamper do Thành làm thì các bạn sẽ test bài tập của hoàng trong component này

    ...Các bạn tự quyết định con số với nhau

  3. in-config-routing.module sẽ là nơi để các bạn khai báo routing cho một loại layout
  4. Ngoài ra các file routing của một module sẽ là nơi các bạn khai báo routing của trang của mình

    Ví dụ: config001 thì khai báo routing cho nó ở file in-config-routing.module

  5. Các thành phần giống nhau trong 1 module thì các tạo component để dùng chung và khai báo nó component đó trong folder components của module đó
  6. Task của mỗi bạn sẽ là tường page trong folder pages của một module (không tạo ra mỗi bạn 1 module)
  7. Đặt tên page tăng dẫn

    Ví dụ: đã có config001 rồi thì component được tạo tiếp theo sẽ là config002

  8. Đây là component test có thể xóa khi các bạn đã rõ thất cả thông tin hoặc ẩn nó đi (không hiển thị trên browser)

  9. Khi khởi động server phải luôn hiển thị ra page nhất định (Không được để trống rồi phải gõ URL )
  10. Tổ chức menu trái như sau:

    Ví dụ: task là làm trang chi tiết đề nghiệp hamper menu trái: Tên group là đề nghiệp hamper -> các function bên trong sẽ là từng task của các bạn.

    Ở cuối trang mà các bạn làm task viết tên của các bạn với đó

  11. NPM When First Pull: npm i --legacy-peer-deps
